# Apple Music BG Loop

Небольшой генератор анимированного фона под картинку
Идея простая: берем картинку, вытаскиваем из нее цветовую структуру и гоняем ее через WebGL как 10 секундный бесшовный луп

## Что делает программа

1. Принимает картинку через `input[type=file]`
2. Сжимает ее до маленького буфера и считает цветовые кластеры
3. Для каждого кластера считает
	 - долю цвета в изображении
	 - центр массы цвета в кадре
	 - концентрацию цвета в пространстве
	 - общую резкость переходов
4. Передает это в шейдер
5. Шейдер строит аморфные пятна с учетом
	 - весов цветов
	 - локальных центров цветов
	 - концентрации и резкости
6. При записи пишет ровно 10 секунд в `webm vp9` чтобы луп замыкался

## Как работает палитра

- Внутри `src/palette.js` используется k-means по пикселям
- Потом близкие кластеры склеиваются чтобы убрать мусорные дубли
- Светлый нейтральный фон приглушается, передний план усиливается
- Дальше все раскладывается в 10 рабочих цветов
- На выходе получается объект
	- `colors[10]`
	- `weights[10]`
	- `anchors[10]`
	- `focus[10]`
	- `sharpness`

## Как работает рендер

- `src/renderer.js` поднимает webgl контекст и держит uniform состояние
- `src/shaders.js` делает noise + domain warp + смешивание цветов
- Каждый цвет двигается вокруг своего `anchor`, амплитуда зависит от `focus`
- Если `focus` высокий, пятно плотнее и не расползается

## режимы в ui

- `движение`
	- `soft` спокойнее
	- `balanced` средний режим
	- `active` более выраженная динамика
- `текстура`
	- `clean` почти без зерна (возможны артефакты)
	- `balanced` умеренно
	- `grain` более заметная пленочность

## запись видео

- используется `canvas.captureStream(60)`
- `MediaRecorder` с `video/webm; codecs=vp9`
- битрейт `25 Mbps`
- длительность `10_000 ms`
- после стопа файл сразу скачивается

## Запуск

1. Открой `index.html` в браузере
2. Загрузи обложку
3. Выстави режимы
4. Нажми `Запись 10s`

## Структура

- `index.html` — UI и подключение скриптов
- `src/main.js` — события и склейка модулей
- `src/palette.js` — анализ картинки
- `src/renderer.js` — WebGL слой
- `src/shaders.js` — GLSL код
- `src/recorder.js` — экспорт видео

Если хочется менять поведение
- баланс цветов лучше трогать в `src/palette.js`
- форму и мягкость переходов в `src/shaders.js`

## Как замыкается луп

- рендер считает `t = elapsed % 10.0` и передает в шейдер
- внутри шейдера тоже `mod(u_time, 10.0)` — на 10-й секунде все возвращается ровно в исходное состояние
- зерно (`grain`) завязано на номер кадра `floor(t * 60.0)`, поэтому при повторном цикле текстура совпадает
- запись стартует из текущего положения и пишет ровно `10 000 ms`, за это время цикл проходит один раз целиком

## Технические детали

- канвас захардкожен `1920×1080`, в css он масштабируется до ширины контейнера через `aspect-ratio: 16/9`
- все модули — iife на `window`, без бандлера — просто открыть `index.html`
- webgl 1, без расширений — работает везде где есть gpu
- запись требует `MediaRecorder` с `video/webm; codecs=vp9` — хром и edge ок, firefox может отказать по кодеку

## Что значат компоненты u_tune

Шейдер принимает `vec4 u_tune`, который собирается из двух селекторов в UI:
- `.x` — амплитуда движения пятен
- `.y` — сила domain warp, чем выше тем живее форма
- `.z` — вклад резкости в мягкость переходов между цветами
- `.w` — количество зерна
